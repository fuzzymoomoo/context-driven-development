# Contributing

Thank you for your interest in Context-Driven Development.

This repository is currently a methodology-first public release. The main job of contributions at this stage is to improve clarity, coherence, and usefulness of the public theory layer.

## Good first contributions

- tighten definitions
- improve examples
- identify contradictions or ambiguity
- improve readability for new readers
- suggest better neutral framing for public documentation

## What to avoid

Please avoid using the repository as a place to:

- dump private workflow notes
- overfit the method to one tool or vendor
- add implementation-specific assumptions without explanation
- introduce public APIs or tooling plans without first clarifying the methodology need

## Contribution guidelines

- Prefer small, focused pull requests.
- Keep terminology consistent across `README.md`, `MANIFESTO.md`, `CONTEXT_MODEL.md`, and `GLOSSARY.md`.
- Keep examples platform-neutral unless a repo maintainer explicitly asks for a stack-specific example.
- If proposing new terminology or doctrine, explain why the current wording is insufficient.

## Discussion priorities

At this stage, the most useful discussion topics are:

- conceptual clarity
- missing distinctions
- better examples
- edge cases for the methodology
- how to keep the public framing portable
